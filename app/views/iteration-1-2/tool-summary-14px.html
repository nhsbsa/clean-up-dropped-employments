What the tool cannot do yet

Null/blank field handling 1.2

Records containing null or blank values in key date or ID fields are not always handled gracefully — some edge cases may produce misleading results or fail to flag the issue correctly.

False positive detection 1.2

Some records that have not been updated may be incorrectly flagged as overlapping.

The tool needs refinement to distinguish genuine corruption from records that are structurally valid but appear anomalous due to missing updates.

New corruption logic 1.2

Emerging patterns — such as status mismatches between tables — are not yet detected. Additional corruption detection logic is required to cover these newly identified failure modes.

Alignment and label consistency 1.2

Several table titles and terminology labels within the tool output are inconsistent — for example, discrepancies between "Service History" and "Service Group" column headings across different sections of the results view.

These will be standardised in the next release to improve readability and reduce user confusion.
